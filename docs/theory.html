Basic Theory

자료구조 시험 전 핵심 복습

Stack, Queue, Array, List, Tree의 정의와 연산, 시간복잡도를 빠르게 확인하세요.

Linear Structure

Stack

마지막에 들어간 데이터가 가장 먼저 나오는 LIFO 구조입니다.

주요 연산

  • push: top에 삽입
  • pop: top 원소 제거
  • top: 가장 위 원소 확인

시간복잡도

  • push: O(1)
  • pop: O(1)
  • top: O(1)

함수 호출, 괄호 검사, 되돌리기처럼 가장 최근 작업을 먼저 처리해야 할 때 사용합니다.

Linear Structure

Queue

먼저 들어간 데이터가 먼저 나오는 FIFO 구조입니다.

주요 연산

  • enqueue: rear에 삽입
  • dequeue: front 원소 제거
  • front: 가장 앞 원소 확인

시간복잡도

  • enqueue: O(1)
  • dequeue: O(1)
  • front: O(1)

대기열, 작업 처리 순서, BFS처럼 먼저 들어온 대상을 먼저 처리해야 하는 상황에 적합합니다.

Sequence Structure

Array

같은 종류의 데이터를 연속된 메모리 공간에 저장하고 index로 접근하는 구조입니다.

주요 연산

  • access: index로 접근
  • set: 특정 위치 값 변경
  • insert / erase: 삽입과 삭제

시간복잡도

  • access: O(1)
  • 끝 삽입: O(1)
  • 중간 삽입/삭제: O(n)

index 접근이 빠른 대신, 중간에 원소를 넣거나 지울 때 뒤 원소들을 이동해야 할 수 있습니다.

Sequence Structure

List

노드들이 링크로 연결되어 순서를 이루는 자료구조입니다.

주요 연산

  • insert: 노드 삽입
  • remove: 노드 삭제
  • search: 값 탐색

시간복잡도

  • 앞 삽입/삭제: O(1)
  • 탐색: O(n)
  • 위치를 아는 삭제: O(1)

크기 변화가 유연하고 삽입/삭제가 편하지만, 특정 index로 바로 접근하기는 어렵습니다.

Hierarchical Structure

Tree

root에서 시작해 부모-자식 관계로 데이터를 계층적으로 표현하는 구조입니다.

주요 연산

  • insert: 노드 추가
  • search: 노드 탐색
  • traversal: preorder / postorder 등 순회

시간복잡도

  • 전체 순회: O(n)
  • 일반 Tree 탐색: O(n)
  • 균형 BST 탐색: O(log n)

파일 시스템, 조직도, 계층 데이터처럼 상하 관계를 표현할 때 유용합니다.
